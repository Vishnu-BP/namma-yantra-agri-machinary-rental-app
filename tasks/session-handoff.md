### Mid-L1 refactor 1: Navigation module (DONE)
Created `src/navigation/` as a new top-level folder (not in CLAUDE.md folder list, but justified — analogous to `src/stores/`):

## Most opinionated design choices (in case the next session wants to revisit)

1. **Navigation guard pattern** — state changes drive redirects; screens never call `router.replace` manually. Cross-group only (same-group is a no-op, prevents loops).
2. **Onboarding rule** — shown whenever no session, not "first launch only." `onboardingStore.reset()` runs on every `SIGNED_OUT`. No AsyncStorage persistence.
3. **PII-aware logging** — log action shapes (booleans, enums, page names) not values. Codified as a new CLAUDE.md rule.
4. **Profile + session loaded together** in `useAuthListener` to avoid the brief race where session is set but profile is still null (would briefly flash role-select for existing users).
5. **OAuth dual-path** — `auth.ts`'s `openAuthSessionAsync` is the standalone-build path; `app/auth/callback.tsx` is the Expo Go path. The first one to set the session wins; the other detects via `getSession()` check and bounces.
